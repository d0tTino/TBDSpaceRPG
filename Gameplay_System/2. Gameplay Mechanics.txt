2. Gameplay Mechanics
=====================

2.1 Core Loop
-------------
The core gameplay loop follows a continuous, real-time simulation model with adjustable time scales:
- **Mission Selection:** Players choose from available missions based on their faction standing, resources, and technological capabilities. Mission availability evolves dynamically as the solar system develops.
- **Real-time Management:** Players continuously manage spacecraft operations and crew dynamics, monitoring systems, adjusting course, and handling interpersonal relationships in both the 3D navigation view and 2D interior view.
- **Time Control:** Players can pause, play at normal speed, or accelerate time (1x, 2x, 5x, 10x) during routine operations, with critical events automatically pausing the simulation to ensure important decisions aren't missed.
- **Mission Completion:** Upon completing mission objectives, players receive immediate rewards and can observe long-term consequences unfold organically in the simulation rather than through forced time skips.
- **Continuous Progression:** Rather than discrete chapters, the game flows continuously, with crew aging, technology advancing, and colonies developing based on player decisions and the passing of time.

2.2 3D Orbital Navigation
-------------------------
Players navigate the solar system using realistic orbital mechanics:
- Trajectory Plotting: Players use a 3D interface to select pre-calculated paths or adjust parameters (e.g., delta-v, burn timing) for custom trajectories.
- Propulsion: Spacecraft use chemical rockets, requiring careful fuel management. Transit times are realistic (e.g., 6-9 months to Mars).
- Tools: The navigation system provides visual aids for transfer windows, fuel consumption, and travel time, ensuring accessibility while maintaining realism.
- Constraints: Players must consider fuel limits, gravity assists, and orbital alignment for efficient travel.

2.3 2D Crew Management
----------------------
Crew management occurs in a 2D top-down view of the ship's interior:
- Crew Roles: Each crew member has a role (e.g., Pilot, Engineer) tied to specific tasks (e.g., navigation, repairs).
- Traits: Crew have traits (e.g., "Hot-headed," "Genius") that influence behavior and task performance.
- Task Assignment: Players assign tasks manually or via schedules, balancing workload and crew needs.
- Status Effects: Morale, health, and relationships affect task efficiency. Low morale or conflicts can delay tasks or cause errors.
- Crew Influence: Crew status directly impacts ship systems (e.g., a stressed Engineer might fail a repair).

2.4 Critical Moments
--------------------
Critical moments are random or scripted events that disrupt missions:
- Triggers: Events like solar flares, micrometeoroid impacts, or crew conflicts occur during transit or at destinations.
- Crew Suggestions: Crew members propose solutions based on their roles and traits (e.g., Pilot suggests course correction, Engineer proposes system fix).
- Player Choice: Players select an option, with success determined by crew status (e.g., skill level, morale).
- Outcomes: Choices can avert disaster, cause damage, or create new challenges, affecting mission success and crew health.

2.5 Difficulty and Failure
--------------------------
The game is designed to be forgiving, with failure leading to alternative paths:
- Failure Scale: Ranges from minor setbacks (e.g., crew injury) to major failures (e.g., mission abort).
- Rival Setbacks: If players struggle, the rival faction (e.g., China) may face random setbacks, balancing difficulty.
- Recovery Options: Players can recover via choices like using a missile to sabotage rivals, altering the story but keeping progression.
- Outcome Carryover: Mission results affect resources, crew composition, and geopolitical balance in future chapters.

- **Faction Influence System:**
  - Player choices impact faction influence values (us_influence, china_influence, independent_coalition)
  - Influence thresholds define geopolitical context:
    - US Dominance (us_influence > 75): Favors individualistic approaches and frontier expansion
    - Chinese Dominance (china_influence > 75): Emphasizes collective goals and centralized planning
    - Coalition Rise (independent_coalition > 60): Promotes collaborative governance and technological sharing
  - Influence scores determine UI aesthetics, recruitment pools, and narrative paths

- **Edge Case: Faction Collapse Scenario**
  - **Chaotic Fragmentation** (all influence scores < 40): When major factions fail to establish dominance and the coalition lacks cohesion, the solar system fragments into isolated pockets of habitation without centralized authority.
    
    - **Aesthetic Impact:** UI adopts a utilitarian patchwork design prioritizing function over form. Color schemes trend toward muted grays and browns with frequent maintenance warnings and system alerts. Multiple languages appear haphazardly across interfaces—English technical terms might appear alongside Mandarin warning labels, Hindi navigation markers, and French distress signals, reflecting the chaotic mix of surviving technologies. Interfaces show visible wear, with flickering displays, static interference during communications, and irregular power fluctuations affecting readability.
    
    - **Environmental Representation:** Habitats display visible degradation—hull patches made from mismatched materials, life support systems jury-rigged with components from different manufacturers, and repurposed military hardware protecting vital resources. Space stations operate independently with minimal communication networks, surrounded by defensive measures against raiders.
    
    - **Narrative Themes:** Storylines focus on basic survival and resource security in a lawless environment. Missions center on maintaining essential supplies, protecting settlements from opportunistic raiders, and securing critical technology before knowledge is lost. Long-term planning becomes nearly impossible as immediate survival takes precedence.
    
    - **Gameplay Impact:** Recruits disproportionately manifest "Self-Reliant," "Resourceful," and "Suspicious" traits (70% probability), with backgrounds in salvage operations, security contracting, and isolated research outposts. Task success rates decrease by specific system:
      - Life support maintenance: -15% success rate due to non-standardized filtration components
      - Propulsion systems: -12% success rate from mismatched fuel injectors and control systems
      - Navigation: -10% accuracy from degraded sensor arrays and unreliable star maps
      - Communications: -20% range and clarity from incompatible transmission protocols
      Crew morale begins 15% lower than baseline, with integration challenges intensified (+10% relationship penalty) due to inherent distrust among survivors from different isolated enclaves.

2.6 Mission Types and Mechanics
-------------------------------
The game features diverse mission types that test player strategy, morality, and leadership while advancing the narrative and influencing faction dynamics. Each mission type offers distinct gameplay challenges that tie into the AI Director system and faction influence mechanics.

- **Exploration Missions:**
  - **Core Mechanics:** Players navigate uncharted sectors of the solar system, plotting trajectories using orbital mechanics (Section 4.8) to reach distant bodies. Gameplay centers on:
    * Scanning operations using ship sensors (directional controls, power allocation)
    * Probe deployment and telemetry analysis (data visualization puzzles)
    * EVA surveys of anomalies (manual control of suited astronauts)
    * Sample collection and analysis (laboratory mini-games)
  
  - **Technical Implementation:**
    ```csharp
    float CalculateExplorationSuccess(CrewMember astronaut, Anomaly target) {
        float baseChance = 50f;
        
        // Skill and trait modifiers
        baseChance += astronaut.GetSkill(SkillType.Science) * 0.5f;
        baseChance += astronaut.GetSkill(SkillType.EVA) * 0.3f;
        
        // Equipment quality
        baseChance += equipment.GetQualityModifier(EquipmentType.Scanner) * 10f;
        
        // Environmental hazards
        if (target.radiationLevel > 200) {
            baseChance -= (target.radiationLevel - 200) * 0.1f;
            astronaut.AddRadiationExposure(target.radiationLevel * 0.5f * timeExposed);
        }
        
        // Faction-specific bonuses
        if (worldState.us_influence > 75 && astronaut.HasTrait("Innovative")) {
            baseChance += 10f; // US technological advantage
        }
        if (worldState.china_influence > 75 && astronaut.HasTrait("Methodical")) {
            baseChance += 10f; // Chinese methodical approach
        }
        
        return Mathf.Clamp(baseChance, 5f, 95f); // Always a chance of success or failure
    }
    ```
  
  - **Challenges:**
    * Environmental hazards (radiation storms: +20 mSv exposure, gravity wells: +15% fuel consumption)
    * Equipment failures (sensor malfunctions: -25% scan accuracy, suit tears: -5% oxygen per minute)
    * Navigation errors (missed burns: +10% fuel consumption, orbit miscalculations: +2 days mission time)
    * Unknown phenomena (magnetic anomalies: communications disruption, micrometeoroid swarms: -5% hull integrity)
  
  - **Narrative Integration:**
    * Discoveries can dramatically increase faction influence (+5-15 points based on significance)
    * First contact with new phenomena may trigger scientific or military response chains
    * The AI Director creates crew confrontations over resource allocation and research priorities
    * Alien artifacts or phenomena become central plot elements in subsequent chapters
    * Crew who make significant discoveries may develop new traits (e.g., "Visionary" or "Obsessive")
    
  - **Chapter Structure Integration:**
    * Exploration missions align with the AI Director's chapter structure (Section 3.4.6):
      * Setup Phase: Mysterious signals or anomalous readings establish exploration objectives
      * Development Phase: Initial sensor sweeps reveal partial information, leading to crew theories/debates
      * Climax Phase: Major discoveries occur at mission peak, often with moral dimensions (e.g., finding evidence of extinct civilization vs. exploitable resources)
      * Resolution Phase: Scientific data processing contextualizes findings within solar system history
    * Discovery timing is carefully paced to match narrative tension curves:
      * Minor findings (geometric formations, unusual mineral compositions) appear in early phases
      * Significant discoveries (artificial structures, biological samples) coincide with the climax
      * Scientific analysis and implications extend into the resolution phase and subsequent chapters
    * The AI Director adjusts anomaly detection probability based on current chapter phase:
      ```csharp
      float CalculateDiscoveryProbability(ScanOperation scan, ChapterPhase currentPhase) {
          float baseChance = scan.GetBaseProbability();
          
          // Adjust based on chapter phase
          switch (currentPhase) {
              case ChapterPhase.Setup:
                  return baseChance * 0.4f; // Reduced early discoveries
              case ChapterPhase.Development:
                  return baseChance * 0.8f; // Moderate hints and minor findings
              case ChapterPhase.Climax:
                  return baseChance * 1.5f; // Dramatically increased major discovery chance
              case ChapterPhase.Resolution:
                  return baseChance * 0.6f; // Follow-up discoveries to contextualize
              default:
                  return baseChance;
          }
      }
      ```

  - **Failure State Handling:** If an exploration mission fails (e.g., equipment malfunction prevents data collection), the AI Director schedules a follow-up event in the Resolution Phase (Section 3.4.6), such as a crew member's guilt (-10% morale) or a rival faction's discovery of the same site (+10 rival influence), setting up future conflicts.

  - **Callback Generation:** The AI Director logs exploration failures in the narrative memory, using them to generate callbacks in future chapters (e.g., a rival faction's discovery in Chapter 1 leads to a territorial dispute in Chapter 3, +15 rival influence), ensuring long-term consequences.

- **Colonization Missions:**
  - **Core Mechanics:** Players establish and manage settlements on distant worlds:
    * Landing site selection (terrain analysis, resource proximity, safety assessment)
    * Habitat construction (module placement, resource pipeline management, expansion planning)
    * Life support management (atmosphere cycling, water purification, food production)
    * Population growth (crew relationships, birth rate factors, genetic diversity management)
  
  - **Technical Implementation:**

    bool EvaluateColonyViability(LandingSite site, ColonyType colonyType) {

        float resourceScore = EvaluateResourceAvailability(site, colonyType);
        float safetyScore = EvaluateSafety(site);
        float habitabilityScore = EvaluateHabitability(site);

        switch (colonyType) {

                totalScore = resourceScore * 0.3f + safetyScore * 0.3f + habitabilityScore * 0.4f;
                break;
            case ColonyType.Mining:
                totalScore = resourceScore * 0.6f + safetyScore * 0.3f + habitabilityScore * 0.1f;

        // Apply faction modifiers

            // US prefers resource-rich sites
            totalScore *= (1f + resourceScore * 0.2f);

  - **Challenges:**

    * Social dynamics (isolation effects: -10% morale per Earth-year, faction tensions: labor disputes)
    * External threats (rival claims: territory disputes, micrometeoroid impacts: -5% structure integrity)
  
  - **Narrative Integration:**
    * Colony success significantly increases faction influence (+15-25 points)
    * Established colonies become critical locations in the solar system's geopolitical landscape
    * The AI Director creates confrontations over colony governance and resource distribution
    * Colonial development affects future mission availability and crew recruitment pools

    bool CalculateWeaponHit(Weapon weapon, Ship target, float distance) {
        float baseHitChance = weapon.baseAccuracy;
        
        // Range modifiers
        float rangeModifier = 1f - (distance / weapon.maxRange) * 0.8f;
        baseHitChance *= rangeModifier;
        
        // Target evasion
        if (target.isPerformingEvasiveManeuvers) {
            baseHitChance *= 0.7f; // 30% reduction when target is evading
        }
        
        // Crew skill (gunner)
        CrewMember gunner = shipSystems.GetSystemOperator(ShipSystem.Weapons);
        if (gunner != null) {
            baseHitChance += gunner.GetSkill(SkillType.Weapons) * 0.03f; // +3% per skill level
        }
        
        // Environmental factors
        if (currentLocation.radiationLevel > 200) {
            baseHitChance *= 0.9f; // Radiation interferes with targeting
        }
        
        // Faction tech advantages
        if (weapon.faction == Faction.US && worldState.us_influence > 75) {
            baseHitChance *= 1.1f; // US targeting systems advantage
        }
        
        // Final calculation
        return Random.Range(0f, 1f) < Mathf.Clamp01(baseHitChance);
    }

    Vector3 CalculateInterceptTrajectory(Ship attacker, Ship target, float projectileSpeed) {
        // Calculate time to intercept based on current positions and velocities
        Vector3 relativePosition = target.position - attacker.position;
        Vector3 relativeVelocity = target.velocity - attacker.velocity;
        
        // Quadratic equation coefficients for intercept time
        float a = Vector3.Dot(relativeVelocity, relativeVelocity) - (projectileSpeed * projectileSpeed);
        float b = 2f * Vector3.Dot(relativeVelocity, relativePosition);
        float c = Vector3.Dot(relativePosition, relativePosition);
        
        // Solve quadratic equation for intercept time
        if (!SolveQuadratic(a, b, c, out float t1, out float t2))
            return Vector3.zero; // No solution possible
            
        // Use earliest positive intercept time
        float interceptTime = (t1 < 0) ? t2 : ((t2 < 0) ? t1 : Mathf.Min(t1, t2));
        
        if (interceptTime < 0)
            return Vector3.zero; // No future intercept possible
            
        // Calculate target's future position
        Vector3 targetFuturePosition = target.position + target.velocity * interceptTime;
        
        // Calculate required velocity vector for projectile
        Vector3 requiredVelocity = (targetFuturePosition - attacker.position) / interceptTime;
        
        return requiredVelocity;
    }
