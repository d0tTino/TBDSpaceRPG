6.1 Glossary
------------
This glossary defines key technical terms used throughout the Gameplay/System Document to ensure clarity and consistency.

- **Crew Traits:** Static personality characteristics (e.g., "Hot-headed," "Genius") that influence a crew member's behavior, task performance, and interactions with other crew members.
- **Orbital Mechanics:** The simulation of spacecraft movement based on real-world physics, including gravity, velocity, and trajectory calculations.
- **Delta-V (Δv):** A measure of the change in velocity required for spacecraft maneuvers, such as course corrections or orbital insertions.
- **Transfer Window:** A specific period when planets are aligned optimally for interplanetary travel, minimizing fuel consumption.
- **Morale:** A dynamic attribute representing a crew member's emotional state, which affects their task efficiency and likelihood of conflicts.
- **Relationship Meter:** A numerical value (e.g., -100 to +100) that tracks the bond between two crew members, influencing their cooperation and task performance.
- **EVA (Extravehicular Activity):** A feature allowing crew members to perform tasks outside the spacecraft, such as repairs or rescues, with success tied to their skills and traits.
- **Aerobraking:** A maneuver using a planet's atmosphere to reduce spacecraft velocity, requiring precise control to avoid damage.
- **Cascading Failures:** A series of system or task failures that compound over time if not addressed, potentially leading to mission-critical issues.

### Kerbal Space Program (KSP) Technical Analysis

KSP serves as a significant technical benchmark for space simulation features, particularly in its approach to orbital mechanics and spacecraft construction.

#### Technical Elements to Adapt:
1. **Orbital Mechanics Implementation**
   - **Patched Conics Approximation:** KSP uses a simplified gravitational model that treats orbital trajectories as connected conic sections. This approach balances realism with computational efficiency.
   ```csharp
   // Our simplified version focusing on performance
   public Vector3 CalculateGravitationalAcceleration(CelestialBody attractingBody, Vector3 position)
   {
       Vector3 directionToBody = attractingBody.position - position;
       float distanceSquared = directionToBody.sqrMagnitude;
       return directionToBody.normalized * (GravitationalConstant * attractingBody.mass / distanceSquared);
   }
   ```

2. **Trajectory Visualization**
   - **Predictive Path Rendering:** KSP uses colored trajectory paths with nodes to visualize orbital maneuvers. We'll implement a similar system but enhance it with:
     - Crew skill influence on trajectory accuracy (±5% deviation based on Pilot skill)
     - Rival ship intercept projections
     - Mission-critical trajectory windows highlighted based on narrative objectives

   public float CalculateMissionSuccessProbability(float availableDeltaV, MissionParameters mission)

        case GameFocus.ExteriorOrbital:

            physicsUpdateInterval = 0.02f;
            // Lower detail for crew simulation
            crewUpdateInterval = 0.5f;
            break;
            
        case GameFocus.InteriorCrew:
            // Lower precision for orbital mechanics
            physicsUpdateInterval = 0.1f;

    Physics.fixedDeltaTime = physicsUpdateInterval;
    crewSimulation.SetUpdateInterval(crewUpdateInterval);

### 6.5.1 Psychological Effects of Long-Duration Spaceflight

Research on the psychological impact of extended space missions reveals several key factors that influence crew performance and well-being:

- **Isolation and Confinement Effects:**
  - Cognitive performance decreases by 20-25% after 6+ months in isolation
  - Sleep quality reduces by 35% in confined environments
  - Communication delays of 20+ minutes significantly impact crew morale
  
- **Countermeasures and Resilience Factors:**
  - Private spaces of 4+ cubic meters reduce psychological stress by 30%
  - Regular Earth communication reduces depression risk by 40-60%
  - Diverse recreational activities improve cognitive performance by 15-20%
  - Virtual reality environments simulating Earth settings reduce homesickness by 45%

- **Team Cohesion:**
  - Mixed-gender crews show 25% better conflict resolution metrics
  - Cultural diversity improves problem-solving but increases initial integration time by 30%
  - Pre-flight relationship building reduces in-mission conflicts by 50%

**Implementation Mechanics:**
```csharp
public float CalculateMoraleImpact(CrewMember crew, MissionParameters mission)
{
    float baseMorale = 0f;
    
    // Communication delay impact
    float commDelay = mission.GetEarthCommunicationDelay(); // in minutes
    if (commDelay > 20f)
    {
        baseMorale -= Mathf.Log(commDelay - 19f) * COMM_DELAY_FACTOR;
    }
    
    // Personal space impact
    float personalSpace = mission.GetPersonalSpacePerCrew(); // in cubic meters
    if (personalSpace < 4f)
    {
        baseMorale -= (4f - personalSpace) * CONFINEMENT_FACTOR;
    }
    else
    {
        baseMorale += Mathf.Min(personalSpace - 4f, 10f) * COMFORT_FACTOR;
    }
    
    // VR access
    if (mission.HasVirtualRealityAccess())
    {
        baseMorale += VR_MORALE_BONUS;
        
        // Diminishing returns over mission duration
        float missionDuration = mission.GetCurrentDuration();
        float diminishingFactor = 1f / (1f + missionDuration / VR_EFFECTIVENESS_HALFLIFE);
        baseMorale -= VR_MORALE_BONUS * (1f - diminishingFactor);
    }
    
    // Apply personal traits modifiers
    if (crew.HasTrait("Introvert"))
    {
        baseMorale += ISOLATION_COMFORT_BONUS;
    }
    if (crew.HasTrait("Extrovert"))
    {
        baseMorale -= ISOLATION_PENALTY * mission.GetCrewCount();
    }
    
    return baseMorale;
}
```

public void UpdateLifeSupportStatus(ShipSystems systems, float deltaTime)
{
    // Calculate base resource recovery rates
    float waterRecoveryRate = systems.GetEfficiency(SystemType.WaterRecycling);
    float oxygenRecoveryRate = systems.GetEfficiency(SystemType.OxygenGeneration);
    float wasteProcessingRate = systems.GetEfficiency(SystemType.WasteProcessing);
    
    // Apply bioregenerative bonuses if present
    if (systems.HasSubsystem(SubsystemType.Greenhouse))
    {
        float greenhouseEfficiency = systems.GetSubsystemEfficiency(SubsystemType.Greenhouse);
        oxygenRecoveryRate += greenhouseEfficiency * PLANT_OXYGEN_CONTRIBUTION;
        
        // Psychological benefit of plants
        CrewManager.ApplyMoraleBonus(PLANT_MORALE_BONUS * greenhouseEfficiency);
    }
    
    // Calculate resource consumption and generation
    int crewCount = CrewManager.GetActiveCrewCount();
    float waterConsumed = crewCount * WATER_CONSUMPTION_PER_CREW * deltaTime;
    float waterRecycled = waterConsumed * waterRecoveryRate;
    float waterLost = waterConsumed - waterRecycled;
    
    float oxygenConsumed = crewCount * OXYGEN_CONSUMPTION_PER_CREW * deltaTime;
    float oxygenRecycled = oxygenConsumed * oxygenRecoveryRate;
    float oxygenLost = oxygenConsumed - oxygenRecycled;
    
    // Update resource banks
    ResourceManager.ConsumeResource(ResourceType.Water, waterLost);
    ResourceManager.ConsumeResource(ResourceType.Oxygen, oxygenLost);
    
    // Check for cascade failure conditions
    bool cascadeRisk = false;
    if (waterRecoveryRate < 0.85f && oxygenRecoveryRate < 0.85f)
    {
        cascadeRisk = true;
        EventManager.TriggerEvent(EventType.CascadeFailureRisk);
    }
    
    // Random failure check
    foreach (SystemType system in systems.GetAllCriticalSystems())
    {
        float reliability = systems.GetReliability(system);
        float failureChance = (1f - reliability) * deltaTime / systems.GetMeanTimeBetweenFailures(system);
        
        if (Random.value < failureChance)
        {
            systems.TriggerSystemFailure(system);
        }
    }
}
```

public float CalculateResourceViability(ResourceDeposit deposit, GameState state)
{
    // Base extraction cost by location
    float baseCost = 0;
    switch (deposit.location.type)
    {
        case LocationType.Lunar:
            baseCost = BASE_LUNAR_EXTRACTION_COST;
            break;
        case LocationType.NEO:
            baseCost = BASE_NEO_EXTRACTION_COST;
            break;
        case LocationType.Mars:
            baseCost = BASE_MARS_EXTRACTION_COST;
            break;
        default:
            baseCost = BASE_OTHER_EXTRACTION_COST;
            break;
    }
    
    // Technology modifier
    float techLevel = state.GetTechnologyLevel(TechField.ResourceExtraction);
    float techModifier = Mathf.Pow(TECH_COST_REDUCTION_BASE, techLevel);
    
    // Concentration modifier
    float concentrationModifier = deposit.concentration / REFERENCE_CONCENTRATION;
    
    // Calculate extraction cost per kg
    float extractionCostPerKg = baseCost * techModifier / concentrationModifier;
    
    // Calculate transport cost to markets
    float transportCost = CalculateTransportCost(deposit.location, state.GetMarketLocations());
    
    // Total cost to market
    float totalCostPerKg = extractionCostPerKg + transportCost;
    
    // Market value of resource
    float marketValue = GetResourceMarketValue(deposit.resourceType, state);
    
    // Return profitability ratio (higher is better)
    return marketValue / totalCostPerKg;
}
```

        float cohesionScore = CalculateCohesionScore();
        
        // Critical threshold for conflict
        if (cohesionScore < 40f)
            return true;
            
        // Chance of conflict increases as cohesion decreases
        if (cohesionScore < 60f)
        {
            float conflictChance = (60f - cohesionScore) * 2f;
            return Random.Range(0f, 100f) < conflictChance;
        }
        
        return false;
    }

        foreach (var religion in _religions)

            }
        }
    }
