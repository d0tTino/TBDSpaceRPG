#### 4.1.1 N-Body Gravitational Interactions

The game implements a hierarchical approach to gravitational simulations:

- **Primary Implementation Strategy:**
  - Patched conics approach for standard gameplay (70% player satisfaction)
  - Optional enhanced physics mode with Barnes-Hut algorithm for enthusiast players
  - Sphere of influence transitions for smooth gravitational changes

- **Performance Optimization:**
  - Full N-body calculations limited to player-controlled vessels and key objects
  - Distance-based culling for gravitational influences
  - Reduced update frequency for distant objects

```csharp
/// <summary>
/// Calculates gravitational acceleration at a given position from multiple bodies
/// </summary>
public Vector3 CalculateGravitationalAcceleration(List<CelestialBody> bodies, Vector3 position)
{
    Vector3 totalAcceleration = Vector3.zero;
    CelestialBody dominantBody = DetermineDominantBody(position, bodies);
    
    // Standard gameplay: use only dominant body for efficiency
    if (!enhancedPhysicsEnabled) {
        Vector3 directionToBody = dominantBody.position - position;
        float distanceSquared = directionToBody.sqrMagnitude;
        return directionToBody.normalized * (G * dominantBody.mass / distanceSquared);
    }
    
    // Enhanced physics: use hierarchical approach
    foreach (CelestialBody body in bodies) {
        // Skip distant bodies with negligible effect using distance-based cutoff
        if (body != dominantBody && !IsSignificantInfluence(body, position)) continue;
        
        Vector3 directionToBody = body.position - position;
        float distanceSquared = directionToBody.sqrMagnitude;
        totalAcceleration += directionToBody.normalized * (G * body.mass / distanceSquared);
    }
    
    return totalAcceleration;
}
```

#### 4.1.2 Non-Spherical Gravity Fields

For realistic orbital motion around planets and irregular bodies:

- **Approximation Methods:**
  - Point-mass model for distant and background celestial bodies
  - J2 term approximation for primary planets (accounts for equatorial bulge)
  - Precomputed gravity maps for irregular bodies like asteroids and comets

```csharp
/// <summary>
/// Calculates gravity including non-spherical effects where appropriate
/// </summary>
public Vector3 CalculateNonSphericalGravity(CelestialBody body, Vector3 position)
{
    // Base two-body gravity
    Vector3 r = position - body.position;
    float r2 = r.sqrMagnitude;
    float r1 = Mathf.Sqrt(r2);
    Vector3 baseAcceleration = -G * body.mass * r / (r2 * r1);
    
    // For irregular bodies or when precision is needed
    if (body.useJ2Effect && highPrecisionMode) {
        // J2 perturbation calculation
        float r5 = r2 * r2 * r1;
        float J2term = 1.5f * body.J2 * body.equatorialRadius * body.equatorialRadius;
        
        // Calculate perturbation components
        float z2 = r.z * r.z;
        Vector3 perturbation = new Vector3(
            r.x * (5f * z2 / r2 - 1f),
            r.y * (5f * z2 / r2 - 1f),
            r.z * (5f * z2 / r2 - 3f)
        );
        
        return baseAcceleration + J2term * perturbation / r5;
    }
    
    return baseAcceleration;
}
```

#### 4.1.3 Atmospheric Drag and Heating

Realistic planetary atmosphere interactions:

- **Simulation Features:**
  - Simplified drag model using exponential atmosphere density profile
  - Heat flux approximation for atmospheric entry effects
  - Visual and audio feedback for player immersion

```csharp
/// <summary>
/// Applies atmospheric drag and heating effects to spacecraft
/// </summary>
public void ApplyAtmosphericEffects(Spacecraft craft, Planet planet, float deltaTime)
{
    // Check if in atmosphere
    float altitude = Vector3.Distance(craft.position, planet.position) - planet.radius;
    if (altitude > planet.atmosphereHeight) return;
    
    // Calculate atmospheric density at altitude
    float relativeAltitude = altitude / planet.atmosphereHeight;
    float density = planet.seaLevelDensity * Mathf.Exp(-planet.atmosphereScaleHeight * relativeAltitude);
    
    // Calculate drag force
    float velocityMagnitude = craft.velocity.magnitude;
    Vector3 dragDirection = -craft.velocity.normalized;
    float dragCoefficient = craft.GetDragCoefficient(craft.attitude, craft.velocity);
    float referenceArea = craft.GetReferenceArea(craft.attitude);
    
    Vector3 dragForce = 0.5f * density * velocityMagnitude * velocityMagnitude * dragCoefficient * referenceArea * dragDirection;
    
    // Apply drag force
    craft.AddForce(dragForce);
    
    // Calculate heating (simplified)
    float heatFlux = HeatMultiplier * density * Mathf.Pow(velocityMagnitude, 3);
    craft.ApplyHeating(heatFlux * deltaTime);
    
    // Visual effects intensity based on heating
    if (craft.visualEffectsEnabled) {
        UpdateAtmosphericEffects(craft, heatFlux);
    }
}
```

#### 4.1.4 Collision Physics

Realistic collision handling for space objects:

- **Implementation Approach:**
  - Rigid body dynamics for standard collisions
  - Pre-fractured models for predetermined break points
  - Particle systems for debris visualization
  - Impact energy calculations for damage modeling

```csharp
/// <summary>
/// Handles collision between spacecraft with realistic physics
/// </summary>
public void HandleSpacecraftCollision(Spacecraft craftA, Spacecraft craftB, CollisionData collision)
{
    // Calculate impact energy
    float relativeSpeed = Vector3.Dot(craftB.velocity - craftA.velocity, collision.normal);
    float impactEnergy = 0.5f * (craftA.mass * craftB.mass / (craftA.mass + craftB.mass)) * relativeSpeed * relativeSpeed;
    
    // Determine damage based on impact energy
    float damageToA = CalculateDamage(impactEnergy, craftA.impactResistance, collision.contactPointOnA);
    float damageToB = CalculateDamage(impactEnergy, craftB.impactResistance, collision.contactPointOnB);
    
    // Apply damage to components
    craftA.ApplyDamageAtLocation(damageToA, collision.contactPointOnA);
    craftB.ApplyDamageAtLocation(damageToB, collision.contactPointOnB);
    
    // Generate debris if damage exceeds threshold
    if (damageToA > craftA.fractureThreshold) {
        GenerateDebris(craftA, collision.contactPointOnA, -collision.normal, impactEnergy);
    }
    
    if (damageToB > craftB.fractureThreshold) {
        GenerateDebris(craftB, collision.contactPointOnB, collision.normal, impactEnergy);
    }
    
    // Apply impact forces (simplified impulse-based physics)
    ApplyCollisionImpulse(craftA, craftB, collision, relativeSpeed);
}
```

#### 4.1.5 Performance-Accuracy Balance

The system employs a level-of-detail approach to balance performance and accuracy:

- **Hierarchical Physics Implementation:**
  | **Object Category** | **Physics Detail** | **Update Frequency** | **Max Count** |
  |---------------------|-------------------|----------------------|---------------|
  | Player Spacecraft   | Full simulation   | Every frame          | 1-5           |
  | Nearby Objects      | High detail       | Every 1-2 frames     | 10-50         |
  | Medium Distance     | Medium detail     | Every 3-5 frames     | 50-200        |
  | Distant Objects     | Position only     | Every 10-30 frames   | 1000+         |

- **Unity-Specific Optimizations:**
  - C# Jobs and Burst compiler for parallel physics calculations
  - ComputeShader implementation for massive N-body simulations
  - GPU instancing for large numbers of similar objects
  - Custom orbital propagation instead of Unity physics for space objects
