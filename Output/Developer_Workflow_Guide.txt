### Windows Terminal
Windows Terminal serves as the central hub for all development activities.

#### Installation
1. Download Windows Terminal from Microsoft Store or GitHub releases
2. Install via standard Windows installer
3. Launch from Start menu

#### Configuration as Default (Optional)
1. Open Windows Terminal > Settings
2. Under Startup, set Default terminal application to Windows Terminal
3. Save configuration

### Git Bash and tmux
The combination of Git Bash with MSYS2-based tmux provides Unix-like terminal multiplexing capabilities.

#### Install Git Bash
1. Download Git from git-scm.com
2. Run installer (path: C:\Program Files\Git)
3. Launch Git Bash from Start menu

#### Install MSYS2 and tmux
1. Download MSYS2 from msys2.org
2. Run installer (path: C:\msys64)
3. Open MSYS2 terminal and update:
   ```
   pacman -Syu
   ```
4. Close and reopen if prompted, then install tmux:
   ```
   pacman -S tmux
   ```

#### Copy tmux to Git Bash
1. Navigate to C:\msys64\usr\bin
2. Copy:
   - tmux.exe
   - msys-event-2.1-7.dll
   - msys-event_core-2.1-7.dll (and other msys-event*.dll files)
3. Paste into C:\Program Files\Git\usr\bin
4. Verify in Git Bash:
   ```
   tmux -V
   ```

#### Configure Git Line Endings
```
git config --global core.autocrlf input
```

### Configuration and Automation

#### Create tmux Session Script
1. In Git Bash:
   ```
   nano ~/tmux-session.sh
   ```
2. Add:
   ```bash
   #!/bin/bash
   export TERM=xterm-256color        # Compatibility with Windows Terminal
   export MSYS=winsymlinks:nativestrict  # Path translation fix
   SESSION_NAME="main"
   if tmux has-session -t $SESSION_NAME 2>/dev/null; then
     tmux attach-session -t $SESSION_NAME
   else
     tmux new-session -s $SESSION_NAME
   fi
   ```
3. Make executable:
   ```
   chmod +x ~/tmux-session.sh
   ```

#### Configure tmux
1. Create/edit ~/.tmux.conf:
   ```
   nano ~/.tmux.conf
   ```
2. Add:
   ```
   # Change prefix to Ctrl+a
   set -g prefix C-a
   unbind C-b
   bind C-a send-prefix

   # Enable mouse support
   set -g mouse on

   # Start window numbering at 1
   set -g base-index 1

   # Split panes
   bind v split-window -h
   bind h split-window -v

   # Performance optimizations
   set -g history-limit 50000       # Larger scrollback
   set -s escape-time 0             # No key delay
   set -g focus-events on           # Editor integration

   # Color support
   set -g default-terminal "xterm-256color"
   set -ga terminal-overrides ",*256col*:Tc"

   # Clipboard integration
   bind -T copy-mode-vi y send-keys -X copy-pipe-and-cancel "clip.exe"
   bind -T copy-mode-vi MouseDragEnd1Pane send-keys -X copy-pipe-and-cancel "clip.exe"
   ```
3. Reload configuration:
   ```
   tmux source-file ~/.tmux.conf
   ```

### Narrative Generation Pipeline

#### Script Structure
The `generate_narrative.py` script uses Hugging Face Transformers with caching:

```python
from transformers import pipeline
import hashlib
import json
import os

class FactionInfluence:
    def __init__(self, us, china, independent):
        self.us_influence = us
        self.china_influence = china
        self.independent_coalition = independent

def get_cached_or_generate(prompt, max_length=100):
    key = hashlib.md5(prompt.encode()).hexdigest()
    cache_file = f"cache/{key}.json"
    
    if os.path.exists(cache_file):
        with open(cache_file, 'r') as f:
            return json.load(f)
    
    generator = pipeline('text-generation', model='distilgpt2')
    result = generator(prompt, max_length=max_length, num_return_sequences=1)
    output = result[0]['generated_text']
    
    os.makedirs("cache", exist_ok=True)
    with open(cache_file, 'w') as f:
        json.dump(output, f)
        
    return output

def generate_event_chain(faction_influence, year, previous_choices):
    """Generate an event chain based on faction influence and game state"""
    prompt = f"""
    In TBD Space Game year {year}, with US influence at {faction_influence.us_influence}, 
    China influence at {faction_influence.china_influence}, and Independent Coalition 
    at {faction_influence.independent_coalition}, generate a diplomatic event 
    where the player's previous choices of {previous_choices} have consequences.
    """
    return get_cached_or_generate(prompt)
```

## 5. TROUBLESHOOTING

### Common Issues and Solutions

#### tmux Not Found
1. Verify msys2 installation
2. Check if tmux.exe and required DLLs are copied to Git Bash bin directory
3. Restart Git Bash

#### Path Translation Issues
1. Ensure the environment variable is set: `export MSYS=winsymlinks:nativestrict`
2. Use forward slashes in paths
3. Escape spaces with backslashes

#### MCP Server Connection Failed
1. Verify server is running: `ps -ef | grep node`
2. Check port availability: `Test-NetConnection -ComputerName localhost -Port 8001 -InformationLevel Quiet`
3. Restart server with `mcpbuild` alias

#### Windows Terminal Profile Missing
1. Open Windows Terminal settings
2. Check JSON configuration
3. Verify paths are correct
4. Restart Windows Terminal

#### Bun Commands Not Working
1. Verify Bun installation: `bun --version`
2. Check PATH environment variable
3. Try using full path to Bun executable

### 7.2 MCP (Machine Control Protocol) Usage

The Machine Control Protocol is a core system for integrating hardware simulation with gameplay systems. Proper MCP integration is critical for maintaining simulation accuracy.

#### 7.2.1 MCP Integration Guidelines

When working with MCP components:

1. **Access Patterns**
   - Always use the MCPManager singleton to access MCP functionality
   - Never directly manipulate hardware simulation state
   - Use provided interfaces in MCPIntegration namespace
   
   ```csharp
   // CORRECT:
   MCPManager.Instance.UpdateSystemStatus(SystemType.LifeSupport, status);
   
   // INCORRECT:
   hardwareSimulation.systems["LifeSupport"].status = status;
   ```

2. **Error Handling**
   - MCP calls must include error handling
   - Use MCPResult return values to check operation success
   - Log MCP failures to the appropriate level (warning or error)
   
   ```csharp
   MCPResult result = MCPManager.Instance.UpdateSystemStatus(SystemType.LifeSupport, status);
   if (!result.Success)
   {
       Debug.LogWarning($"MCP update failed: {result.ErrorMessage}");
       // Handle failure appropriately
   }
   ```

3. **Performance Considerations**
   - Batch MCP updates when possible using UpdateBatch()
   - Avoid polling MCP status in Update() methods
   - Subscribe to MCP events instead of continuous checking
   
   ```csharp
   // Subscribe to MCP events
   void OnEnable()
   {
       MCPManager.Instance.OnSystemStatusChanged += HandleSystemStatusChanged;
   }
   
   void OnDisable()
   {
       MCPManager.Instance.OnSystemStatusChanged -= HandleSystemStatusChanged;
   }
   ```
