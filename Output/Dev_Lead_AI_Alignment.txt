## Code Standards

1. **C# Conventions**
   - Use PascalCase for public members and camelCase for private members
   - XML documentation for all public methods
   - Unity-specific patterns (e.g., MonoBehaviour lifecycle methods)
   - Preference for composition over inheritance

2. **Unity Best Practices**
   - ScriptableObjects for configuration data
   - Dependency injection for manager references
   - Event-based communication between systems
   - Resource pooling for frequently instantiated objects

3. **Performance Considerations**
   - Minimize GameObject instantiation at runtime
   - Use non-allocating code patterns in hot paths
   - Batch operations where possible
   - Implement LOD (Level of Detail) for complex visualizations

The task requires implementing a new method in ShipCustomizationManager to handle thrust upgrades. This needs to interface with the MCP system and update ship performance parameters.

## Current State Assessment

## Implementation Strategy
I'll break this into 3 discrete steps focusing on: method implementation, MCP integration, and testing.

## Step 1: Implement UpgradeThrust Method
Add the following method to ShipCustomizationManager.cs:
```csharp
public void UpgradeThrust(float upgradeAmount)
{
    // Validate input
    if (upgradeAmount <= 0f)
    {
        Debug.LogError("Upgrade amount must be positive");
        return;
    }
    
    // Apply upgrade
    currentShipStats.thrustCapability += upgradeAmount;
    
    // Apply any constraints
    currentShipStats.thrustCapability = Mathf.Min(
        currentShipStats.thrustCapability,
        maxShipStats.thrustCapability
    );
    
    // Notify listeners
    OnShipStatsChanged?.Invoke(currentShipStats);
    
    Debug.Log($"Ship thrust upgraded by {upgradeAmount}. New thrust: {currentShipStats.thrustCapability}");
}
```
CHECKPOINT: Verify the method compiles without errors and follows the established code style.

## Step 2: Create MCP Integration
Register a new menu item in ShipUpgradeMenuHelperV2.cs:
```csharp
[MenuItem("MCP/Ship/V2/Upgrade Thrust")]
public static void UpgradeThrust()
{
    // Locate ship customization manager
    ShipCustomizationManager manager = FindObjectOfType<ShipCustomizationManager>();
    if (manager == null)
    {
        Debug.LogError("ShipCustomizationManager not found in scene");
        return;
    }
    
    // Apply standard upgrade amount

    manager.UpgradeThrust(STANDARD_THRUST_UPGRADE);
    
    Debug.Log("MCP Command: Ship thrust upgraded");
}
```
CHECKPOINT: Verify the menu item appears in Unity's MCP menu and can be triggered manually.

## Step 3: Test Integration

CHECKPOINT: Verify that the thrust upgrade is properly applied and the ship statistics are updated.

## Recent Strategic Design Shifts

### Continuous Gameplay Model (Updated April 3, 2025)
The project has pivoted from a chapter-based structure with forced time skips to a continuous gameplay model with optional time acceleration, more aligned with Paradox-style grand strategy games. This fundamental shift affects several key aspects of our development:

#### Narrative Integration
- Maintain core story elements and branching narratives while allowing them to unfold organically
- Replace discrete chapter transitions with dynamic event triggers based on player actions and time progression
- Ensure critical narrative moments still occur but are integrated into the continuous simulation

#### Technical Implications
- Refactor simulation systems to support continuous time management with various acceleration rates
- Implement event-driven architecture to handle critical decision points that pause time acceleration
- Develop robust state tracking for long-term evolution of colonies, crew aging, and faction dynamics
- Create UI elements for time control (pause, 1x, 2x, 5x, 10x speeds)

#### AI Director Enhancements
- Expand the AI Director's role to dynamically generate appropriate missions and events based on the continuous world state
- Implement more sophisticated faction AI to make decisions in parallel with the player over continuous time
- Develop systems for generating procedural narrative content that remains coherent over long time periods

This change aligns with our goal of creating a more immersive, player-driven experience while maintaining the core narrative themes and mechanical depth of our original vision. The continuous model provides greater agency and better reflects the organic nature of space exploration and colonization.
