### 1.2 Core Components

#### 1.2.1 Godot Client
- Godot 4.2 with .NET
- C# (Godot .NET) and GDScript for game logic
- WebSocket client for MCP communication
- Automatic glTF import from `Assets_glTF/`
- Custom Godot editor tools for development

#### 1.2.2 MCP Server
- Bun runtime environment
- WebSocket server (8001 port)
- JSON-based protocol
- Menu-driven command system

#### 1.2.3 Game Systems
- Ship customization and management
- Crew simulation and relationships
- Orbital mechanics and navigation
- Time skip and generational progression

#### 1.2.4 AI Services
- Narrative content generation
- Faction relationship simulation
- Dynamic event generation
- Character development
### 1.3 Asset Pipeline
- Export assets as `.gltf` from Unity or Blender
- Store exports under `Assets_glTF/` at the project root
- The Godot project auto-imports the glTF files and generates scenes
- Scenes are instanced by the Godot client at runtime

## 2. System Specifications

### 2.1 Communication Protocol

#### 2.1.1 WebSocket Configuration
```json
{
    "port": 8001,
    "maxPayloadSize": "1MB",
    "heartbeatInterval": 30000,
    "reconnectAttempts": 3,
    "bufferSize": {
        "send": "256KB",
        "receive": "512KB"
    }
}
```

#### 2.1.2 Message Format
```json
{
    "messageId": "uuid-v4",
    "type": "command|response|event",
    "menuPath": "Category/Subcategory/Action",
    "parameters": {},
    "timestamp": "ISO-8601",
    "version": "1.0"
}
```

### 2.2 Performance Requirements

#### 2.2.1 Client Performance
- Target FPS: 60 (Godot 4.2 Vulkan)
- Maximum memory usage: 4GB
- Load time: <30 seconds
- Asset streaming: Dynamic glTF scenes

#### 2.2.2 Server Performance
- Maximum concurrent connections: 100
- Response time: <50ms
- Memory usage: <1GB
- CPU usage: <30% single core

#### 2.2.3 AI System Performance
- Content generation: <2s
- Memory usage: <750MB per model
- Cache hit rate: >70%
- Batch processing: <5s for 32 items

### 2.3 Data Storage

#### 2.3.1 Save Game Format
```json
{
    "version": "1.0",
    "timestamp": "ISO-8601",
    "gameState": {
        "worldTime": "long",
        "playerStats": {},
        "shipState": {},
        "crewState": {},
        "relationships": [],
        "missions": []
    },
    "checksums": {
        "gameState": "sha256",
        "relationships": "sha256"
    }
}
```

#### 2.3.2 Content Cache Structure
```
/cache
  /dialogue
    /{hash}.json
  /events
    /{hash}.json
  /characters
    /{hash}.json
  /metadata
    cache_index.json

```
### 2.4 Asset Pipeline
- glTF exports stored in `Assets_glTF/`
- Godot imports the files and produces `.tscn` scenes
- Scenes are referenced by name in C# scripts
- Imports refresh automatically when assets change

## 4. Development Workflow

### 4.1 Build Process
1. Generate AI components
2. Validate generated code
3. Integrate with Unity project
4. Run automated tests
