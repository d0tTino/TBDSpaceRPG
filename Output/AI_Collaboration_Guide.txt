### Workflow Guidelines

#### 1. Content Request Format

When providing narrative generation prompts, follow these patterns:

```python
# Crew Dialogue Generation
generate_dialogue(
    character_id="engineer_sofia",
    context="reacting_to_critical_system_failure",
    emotional_state="stressed",
    relationship_context={"captain": "respectful", "pilot": "conflicted"},
    tone="technical_but_urgent"
)

# Event Description Generation
generate_event_description(
    event_type="asteroid_field_encounter",
    severity="moderate",
    crew_involved=["pilot", "scientist"],
    ship_systems_affected=["sensors", "navigation"],
    tone="tense_scientific"
)

# Character Background Generation
generate_character_background(
    role="medical_officer",
    origin="martian_colony",
    personality_traits=["compassionate", "perfectionist", "haunted"],
    generation=2,  # 0=Earth-born, 1=first-gen colonist, 2+=colony-born
    narrative_hooks=["lost_family_in_colony_disaster", "pioneered_genetic_therapy"]
)
```

#### 2. FactionInfluence System

The FactionInfluence system dynamically models political tensions between major factions and generates appropriate events and dialogue. Usage examples:

```python
# Initialize faction relationship matrix
faction_system = FactionInfluenceSystem()
faction_system.set_relationship("united_states", "china", -65)  # -100 to 100 scale
faction_system.set_relationship("united_states", "europe", 40)
faction_system.set_relationship("china", "india", -30)

# Generate faction-specific event
event = faction_system.generate_event(
    primary_faction="united_states",
    secondary_faction="china",
    event_category="diplomatic",
    player_alignment=0.7,  # 0-1 scale, 1=fully aligned with primary faction
    tension_threshold=0.6  # 0-1 scale, events only trigger above threshold
)

# Generate faction-influenced dialogue
dialogue = faction_system.generate_dialogue(
    speaker_faction="united_states",
    subject_faction="china",
    context="resource_dispute",
    speaker_role="diplomat",
    player_reputation_with_speaker=0.8  # 0-1 scale
)
```

**Performance Tip:** Target inference time of <2s on i5-12400 systems when generating faction events. For more detailed specifications on event integration and faction relationships, see Narrative Design Document (NDD Section 1.3, "Event Integration").

#### 3. Content Caching System

The caching system ensures content availability during offline play and optimizes performance:

```python
# Cache initialization
content_cache = ContentCache(cache_path="~/TBD_Space_Game_scripts/content_cache")

# Cache narrative content
cache_key = content_cache.store(
    content_type="dialogue",
    content=dialogue_text,
    metadata={
        "character": "engineer_sofia",
        "context": "system_failure",
        "generated_at": datetime.now().isoformat(),
        "model_version": "gpt-neo-1.3B"
    }
)

# Retrieve cached content
cached_dialogue = content_cache.retrieve(
    content_type="dialogue",
    filter_criteria={
        "character": "engineer_sofia",
        "context": "system_failure"
    },
    fallback_generator=generate_dialogue,  # Function to call if cache miss
    allow_online_fallback=True  # Whether online generation is allowed
)

# Cache maintenance
content_cache.prune(
    older_than_days=30,
    except_types=["character_backgrounds", "critical_events"],
    min_entries_to_keep=5
)
```

#### 4. Human-in-the-Loop Review Process

Critical narrative content undergoes human review before integration:

```python
# Submit content for review
review_id = review_system.submit(
    content=generated_text,
    content_type="major_plot_event",
    priority="high",
    suggested_edits="Consider toning down political implications"
)

# Check review status
status = review_system.check_status(review_id)
# Returns: {"status": "pending|approved|rejected|modified", "reviewer": "username", "notes": "..."}

# Apply reviewed content
if status["status"] == "approved" or status["status"] == "modified":
    final_content = review_system.get_final_content(review_id)
    content_manager.integrate(final_content, destination="plot_events_db")
```

### Offline Generation Guidelines

When developing content generators for offline use:

1. **Model Size Constraints:**
   - Keep models under 500MB total size
   - Prefer quantized models (16-bit or 8-bit precision)
   - Target inference time < 2 seconds on mid-range hardware

2. **Content Diversity:**
   - Create generator-specific seed management to avoid repetition
   - Implement template-guided generation for consistent formatting
   - Use mixtures-of-experts approach for varied but coherent output

3. **Integration with Game Systems:**
   - Content generators should accept game state parameters
   - Output should include metadata for game system integration
   - Support for regeneration with same inputs if result is unsuitable

## Troubleshooting Guide

### MCP Connection Issues
1. **Problem**: Unable to connect to MCP server
   **Diagnostic**: Check if server is running and port is available
   **Solution**: Use `Test-NetConnection -ComputerName localhost -Port 8001 -InformationLevel Quiet` to check the port, then restart server

2. **Problem**: Connection errors in Unity console
   **Diagnostic**: WebSocket connection refused or timeout
   **Solution**: Ensure correct port, check Node.js is running, verify firewall settings

3. **Problem**: GUI layout errors in editor
   **Diagnostic**: Error about invalid GUI layout state
   **Solution**: These are display-only issues, can be ignored, but restarting Unity clears them

### Performance Optimization Guidelines

#### 1. Model Quantization
- Use 8-bit quantized models for 40-50% memory reduction
- Target models under 2GB total memory usage
- Pre-load frequently used model components

#### 2. Batch Processing
- Batch similar requests (e.g., multiple dialogues)
- Optimal batch size: 16-32 items
- Process batches during loading screens

#### 3. Cache Management
```python
# Implement tiered caching
class ContentCache:
    def __init__(self):
        self.memory_cache = {}  # Fast, limited size
        self.disk_cache = {}    # Slower, persistent
        self.memory_limit = 100MB
    
    def get(self, key):
        # Check memory cache first
        if key in self.memory_cache:
            return self.memory_cache[key]
        
        # Check disk cache
        if key in self.disk_cache:
            result = self.disk_cache[key]
            self._promote_to_memory(key, result)
            return result
        
        return None
    
    def _promote_to_memory(self, key, value):
        if self._get_memory_usage() > self.memory_limit:
            self._evict_oldest()
        self.memory_cache[key] = value
```

#### 2. Performance Logging
```python
# Track generation times
@contextmanager
def timing_context(operation_name):
    start_time = time.time()
    try:
        yield
    finally:
        duration = time.time() - start_time
        logging.info(f"{operation_name} took {duration:.2f}s")
```

#### 3. Error Recovery
```python
# Implement fallback generation
def generate_with_fallback(prompt, primary_model, fallback_model):
    try:
        with timing_context("primary_generation"):
            return primary_model.generate(prompt)
    except (RuntimeError, OutOfMemoryError):
        logging.warning("Primary model failed, falling back to smaller model")
        with timing_context("fallback_generation"):
            return fallback_model.generate(prompt)
```
